Aprende SQL con ejemplos claros y sintaxis resaltada

Conceptos esenciales y recetas listas para usar en MySQL/MariaDB y compatibles con otros motores. Todo en un solo lugar.

⚡ Resumen práctico: crea ↔ consulta ↔ filtra ↔ actualiza ↔ borra

📘 1) ¿Qué es SQL?

Lenguaje estándar para gestionar datos en sistemas relacionales.

Con SQL defines estructuras (tablas, índices), manipulas datos (insertar, consultar, actualizar, borrar) y controlas permisos/transacciones. Piensa en SQL como el "idioma" para hablar con tu base de datos.

-- Consulta básica SELECT * FROM productos; -- Inserción mínima INSERT INTO productos(nombre, precio) VALUES ('Cuaderno', 4500);

🧭 2) Partes del SQL: DDL, DML, DCL, TCL

Categorías de comandos según su propósito.

  • DDL (Data Definition Language): CREATE, ALTER, DROP — definen estructuras.
  • DML (Data Manipulation Language): SELECT, INSERT, UPDATE, DELETE.
  • DCL (Data Control Language): GRANT, REVOKE — permisos/roles.
  • TCL (Transaction Control Language): COMMIT, ROLLBACK, SAVEPOINT.
  • Extras útiles: EXPLAIN (plan de ejecución), DESCRIBE (metadatos).

🧱 3) Crear una base de datos y seleccionarla

CREATE DATABASE empresa_db CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci; USE empresa_db; -- selecciona el esquema activo

📋 4) Crear una tabla por SQL

CREATE TABLE cliente ( id_cliente BIGINT PRIMARY KEY AUTO_INCREMENT, nombre VARCHAR(160) NOT NULL, doc_tipo VARCHAR(4) NOT NULL, doc_numero VARCHAR(30) NOT NULL, ciudad VARCHAR(60) NOT NULL, email VARCHAR(120), UNIQUE KEY UX_doc (doc_tipo, doc_numero) );

➕ 5) Insertar datos por SQL (dos formas)

-- Forma 1: columnas explícitas INSERT INTO cliente(nombre, doc_tipo, doc_numero, ciudad, email) VALUES ('RápidoExpress S.A.S.', 'NIT', '900123456-1', 'Bogotá', 'contacto@empresa.com'); -- Forma 2: múltiples filas INSERT INTO cliente(nombre, doc_tipo, doc_numero, ciudad) VALUES ('Ana Pérez','CC','10203040','Medellín'), ('Carlos Ruiz','CC','55667788','Cali');

🔎 6) Consultar todos los datos

SELECT * FROM cliente;

🎯 7) Filtrar con WHERE

SELECT nombre, ciudad FROM cliente WHERE ciudad = 'Bogotá';

🧹 8) Eliminar duplicados con DISTINCT

SELECT DISTINCT ciudad FROM cliente ORDER BY ciudad;

✏️ 9) Actualizar con UPDATE

UPDATE cliente SET email = 'nuevo@dominio.com' WHERE id_cliente = 1;

🗑️ 10) Borrar con DELETE

DELETE FROM cliente WHERE ciudad <> 'Bogotá';

⚖️ 11) Operadores lógicos AND / OR

SELECT * FROM cliente WHERE ciudad='Bogotá' AND doc_tipo='NIT' OR nombre LIKE 'A%';

Consejo: usa paréntesis para dejar clara la precedencia. Ej.: (ciudad='Bogotá' AND doc_tipo='NIT') OR nombre LIKE 'A%'

📏 12) Rangos con BETWEEN

SELECT * FROM pedido WHERE total BETWEEN 50000 AND 100000;

🔍 13) Búsquedas con LIKE

SELECT * FROM cliente WHERE nombre LIKE '%Express%';

Usa % como comodín de múltiples caracteres y _ para uno solo.

🎛️ 14) Múltiples valores con IN

SELECT * FROM cliente WHERE ciudad IN ('Bogotá','Cali','Medellín');

🧪 5 ejemplos rápidos

  • Top 10 productos caros:
    SELECT nombre, precio FROM productos ORDER BY precio DESC LIMIT 10;
  • Clientes con correo corporativo:
    SELECT * FROM cliente WHERE email LIKE '%@empresa.com';
  • Ventas del mes actual:
    SELECT * FROM ventas WHERE fecha BETWEEN DATE_FORMAT(CURDATE(), '%Y-%m-01') AND LAST_DAY(CURDATE());
  • Usuarios de varias ciudades:
    SELECT nombre, ciudad FROM usuario WHERE ciudad IN ('Bogotá','Cali','Barranquilla');
  • Actualizar estado masivo:
    UPDATE pedido SET estado='ENVIADO' WHERE estado='PAGADO' AND fecha <= CURDATE();
