🎯 1) SELECT WHERE Básico: Explicación

Filtra registros específicos basados en condiciones simples.

En ingeniería industrial, WHERE es clave para analizar subconjuntos de datos, como producción defectuosa en una máquina específica. Evalúa condiciones booleanas (verdadero/falso) y devuelve solo filas que las cumplan. Operadores comunes: = (igual), <> (diferente), >, <, etc. Es eficiente para optimizar consultas y evitar procesar datos innecesarios, ahorrando tiempo en reportes de calidad.

💻 1) Sintaxis Básica

SELECT columnas FROM tabla WHERE condicion; -- Ej: Filtrar máquina M1 en turno 1 SELECT * FROM produccion WHERE maquina = 'M1' AND turno = 1;

🧪 5 Ejemplos Prácticos

🔍 Producción en máquina M1

SELECT * FROM produccion WHERE maquina = 'M1';

⚠️ Turnos con más de 5 defectos

SELECT fecha, turno, defectos FROM produccion WHERE defectos > 5;

📈 Piezas superiores a 150

SELECT maquina, piezas FROM produccion WHERE piezas > 150;

💰 Costos menores a $200

SELECT * FROM produccion WHERE costo_hora < 200;

📅 Fechas específicas (Oct 2)

SELECT * FROM produccion WHERE fecha = '2025-10-02';

🔥 Reto: Identifica turnos ineficientes

Escribe una consulta WHERE para encontrar turnos donde defectos > piezas * 0.05 (5% tolerancia). ¿Cuántos hay?

🧹 2) DISTINCT: Explicación

Elimina duplicados de los resultados para obtener valores únicos.

Útil en auditorías industriales para listar máquinas únicas o turnos sin repeticiones. DISTINCT actúa sobre todas las columnas seleccionadas; si hay duplicados exactos, los suprime. Mejora la claridad en dashboards de KPI, evitando inflar conteos. Nota: es más lento en tablas grandes, úsalo con moderación.

💻 2) Sintaxis Básica

SELECT DISTINCT columnas FROM tabla; -- Ej: Máquinas únicas SELECT DISTINCT maquina FROM produccion;

🧪 5 Ejemplos Prácticos

🏭 Máquinas únicas

SELECT DISTINCT maquina FROM produccion;

⏰ Turnos únicos

SELECT DISTINCT turno FROM produccion;

📅 Fechas únicas

SELECT DISTINCT fecha FROM produccion ORDER BY fecha;

🔗 Combinación máquina-turno

SELECT DISTINCT maquina, turno FROM produccion;

💰 Costos únicos

SELECT DISTINCT costo_hora FROM produccion;

🔥 Reto: Unicidad en defectos

Lista valores únicos de defectos y cuenta cuántos son cero. ¿Qué implica para calidad?

📏 3) LIMIT: Explicación

Restringe el número de filas devueltas, ideal para muestreo rápido.

En análisis industrial, LIMIT acelera pruebas o vistas previas de grandes datasets de producción. Especifica un tope (ej: top 5 peores días). Combínalo con ORDER BY para resultados relevantes. No afecta el total real, solo la salida; perfecto para paginación en apps de monitoreo.

💻 3) Sintaxis Básica

SELECT columnas FROM tabla LIMIT numero_filas; -- Ej: Primeros 3 registros SELECT * FROM produccion LIMIT 3;

🧪 5 Ejemplos Prácticos

📋 Primeras 5 producciones

SELECT * FROM produccion LIMIT 5;

📉 Últimas 3 (con ORDER BY)

SELECT * FROM produccion ORDER BY id DESC LIMIT 3;

🏆 Top 4 por piezas

SELECT * FROM produccion ORDER BY piezas DESC LIMIT 4;

🔧 Muestreo de 2 máquinas

SELECT * FROM produccion WHERE maquina = 'M1' LIMIT 2;

🚨 Solo 1 con más defectos

SELECT * FROM produccion ORDER BY defectos DESC LIMIT 1;

🔥 Reto: Muestreo rápido

Limita a 3 los turnos con costo > $210. ¿Son viables económicamente?

🔄 4) ORDER BY: Explicación

Ordena resultados por una o más columnas, ascendente (ASC) o descendente (DESC).

Esencial para priorizar en ingeniería: ordena por eficiencia (piezas/día) o urgencia (defectos altos primero). Por defecto ASC; usa DESC para reverso. Múltiples columnas permiten orden secundario. Optimiza toma de decisiones en lean manufacturing al visualizar tendencias claras.

💻 4) Sintaxis Básica

SELECT columnas FROM tabla ORDER BY columna [ASC|DESC]; -- Ej: Ordenar por piezas descendente SELECT * FROM produccion ORDER BY piezas DESC;

🧪 5 Ejemplos Prácticos

📅 Por fecha ascendente

SELECT * FROM produccion ORDER BY fecha ASC;

🚨 Por defectos descendente

SELECT * FROM produccion ORDER BY defectos DESC;

🔗 Por piezas y luego turno

SELECT * FROM produccion ORDER BY piezas DESC, turno ASC;

💰 Costos bajos primero

SELECT maquina, costo_hora FROM produccion ORDER BY costo_hora ASC;

🏭 Turnos por máquina

SELECT * FROM produccion ORDER BY maquina ASC, turno DESC;

🔥 Reto: Orden de prioridad

Ordena por defectos DESC y luego por costo_hora ASC. Identifica el más problemático.

📋 5) IN y NOT IN: Explicación

Filtra múltiples valores en una lista; NOT IN invierte la condición.

Práctico para ingenieros al segmentar por categorías, como máquinas específicas o turnos prioritarios. IN reemplaza múltiples OR, más legible y eficiente. NOT IN excluye grupos, útil para aislar anomalías en control de calidad. Limita listas largas para rendimiento.

💻 5) Sintaxis Básica

SELECT columnas FROM tabla WHERE columna IN (val1, val2, ...); -- Ej: Máquinas M1 o M2 SELECT * FROM produccion WHERE maquina IN ('M1', 'M2'); -- NOT IN: Excluir M1 SELECT * FROM produccion WHERE maquina NOT IN ('M1');

🧪 5 Ejemplos Prácticos

⏰ Turnos 1 y 3

SELECT * FROM produccion WHERE turno IN (1, 3);

❌ Excluir máquina M2

SELECT * FROM produccion WHERE maquina NOT IN ('M2');

📅 Fechas Oct 1 y 3

SELECT piezas FROM produccion WHERE fecha IN ('2025-10-01', '2025-10-03');

🚫 Turnos no 2

SELECT * FROM produccion WHERE turno NOT IN (2);

🏭 Máquinas específicas

SELECT costo_hora FROM produccion WHERE maquina IN ('M1', 'M2');

🔥 Reto: Segmentación

Usa NOT IN para excluir turnos con defectos > 5. ¿Mejora el promedio de piezas?

📐 6) BETWEEN: Explicación

Filtra rangos inclusivos (de A a B, incluye A y B).

Incluye en análisis de series temporales industriales, como producción en un período o rangos de costos viables. Sintaxis: columna BETWEEN valor_min AND valor_max. Funciona con fechas, números; equivalente a >= min AND <= max. Ideal para forecasting de inventarios.

💻 6) Sintaxis Básica

SELECT columnas FROM tabla WHERE columna BETWEEN valor_min AND valor_max; -- Ej: Costos entre 195 y 210 SELECT * FROM produccion WHERE costo_hora BETWEEN 195 AND 210;

🧪 5 Ejemplos Prácticos

💰 Costos 200-210

SELECT * FROM produccion WHERE costo_hora BETWEEN 200 AND 210;

📦 Piezas 140-160

SELECT maquina, piezas FROM produccion WHERE piezas BETWEEN 140 AND 160;

✅ Defectos 0-5

SELECT * FROM produccion WHERE defectos BETWEEN 0 AND 5;

📅 Fechas Oct 2-4

SELECT * FROM produccion WHERE fecha BETWEEN '2025-10-02' AND '2025-10-04';

⏰ Turnos 1-2

SELECT turno, piezas FROM produccion WHERE turno BETWEEN 1 AND 2;

🔥 Reto: Rango óptimo

Filtra piezas BETWEEN 150 AND 170 y calcula manualmente el promedio de defectos. ¿Es bajo?

🔍 7) LIKE: Explicación

Búsquedas con patrones usando comodines % (cualquier cadena) y _ (un carácter).

En contextos industriales, LIKE es ideal para buscar códigos de máquinas parciales, descripciones de errores o fechas aproximadas. % coincide con cero o más caracteres; _ con exactamente uno. Úsalo para auditorías flexibles, como identificar todas las máquinas 'M1%' (variantes de M1). Combínalo con WHERE para filtrados dinámicos en logs de producción, mejorando el troubleshooting.

💻 7) Sintaxis Básica

SELECT columnas FROM tabla WHERE columna LIKE 'patron'; -- Ej: Máquinas que empiecen con 'M1' SELECT * FROM produccion WHERE maquina LIKE 'M1%';

🧪 5 Ejemplos Prácticos

🏭 Máquinas empezando con M1

SELECT * FROM produccion WHERE maquina LIKE 'M1%';

📅 Fechas en octubre temprano (01-09)

SELECT * FROM produccion WHERE fecha LIKE '2025-10-0_';

🚨 Máquinas terminando en 2

SELECT maquina, defectos FROM produccion WHERE maquina LIKE '%2';

🔤 Máquinas con cualquier carácter (M_)

SELECT * FROM produccion WHERE maquina LIKE 'M_'; -- M1 o M2

📦 Piezas con dígito específico (ej: 15_)

SELECT * FROM produccion WHERE CAST(piezas AS CHAR) LIKE '15_'; -- 150-159

🔥 Reto: Búsqueda de patrones

Usa LIKE para encontrar producciones donde maquina termine en '1' y defectos > 5. ¿Hay patrones de fallos?

📈 8) Funciones de Agrupamiento Básico: Explicación

Resumen por grupos: COUNT (cuenta), SUM (suma), AVG (promedio), MIN/MAX (extremos).

Cruciales para métricas industriales como OEE (eficiencia). GROUP BY agrupa filas por columna; aplica funciones a cada grupo. COUNT(*) cuenta filas, SUM suma valores, AVG promedia, MIN/MAX extremos. Úsalos para KPIs como producción total por máquina o tasa de defectos promedio.

💻 8) Sintaxis Básica

SELECT columna_agrupacion, funcion(columna) FROM tabla GROUP BY columna_agrupacion; -- Ej: Total piezas por máquina SELECT maquina, SUM(piezas) AS total_piezas FROM produccion GROUP BY maquina;

🧪 5 Ejemplos Prácticos (uno por función)

🔢 COUNT: Registros por máquina

SELECT maquina, COUNT(*) AS registros FROM produccion GROUP BY maquina;

➕ SUM: Total piezas por turno

SELECT turno, SUM(piezas) AS total_piezas FROM produccion GROUP BY turno;

📊 AVG: Promedio defectos por fecha

SELECT fecha, AVG(defectos) AS prom_defectos FROM produccion GROUP BY fecha;

⬇️ MIN: Costo mínimo por máquina

SELECT maquina, MIN(costo_hora) AS min_costo FROM produccion GROUP BY maquina;

⬆️ MAX: Máx piezas por turno

SELECT turno, MAX(piezas) AS max_piezas FROM produccion GROUP BY turno;

🔥 Reto: KPI por máquina

Calcula AVG(piezas) y SUM(defectos) GROUP BY maquina. ¿Cuál es más eficiente?

🔍 9) Agrupamiento con HAVING: Explicación

Filtra grupos después de GROUP BY, similar a WHERE pero para agregados.

Permite condiciones en resúmenes, como grupos con promedio > umbral. En industrial, filtra máquinas con bajo rendimiento (ej: AVG(piezas) < 140). WHERE filtra filas pre-agrupación; HAVING post. Combínalos para análisis profundos de eficiencia operativa.

💻 9) Sintaxis Básica

SELECT columna_agrupacion, funcion(columna) FROM tabla GROUP BY columna_agrupacion HAVING condicion_en_agregado; -- Ej: Máquinas con total > 500 piezas SELECT maquina, SUM(piezas) AS total FROM produccion GROUP BY maquina HAVING total > 500;

🧪 5 Ejemplos Prácticos

📦 Total piezas > 600 por máquina

SELECT maquina, SUM(piezas) FROM produccion GROUP BY maquina HAVING SUM(piezas) > 600;

📊 Promedio defectos < 4 por turno

SELECT turno, AVG(defectos) FROM produccion GROUP BY turno HAVING AVG(defectos) < 4;

🔢 COUNT > 2 por fecha

SELECT fecha, COUNT(*) FROM produccion GROUP BY fecha HAVING COUNT(*) > 2;

💰 SUM costos > 1000 por máquina

SELECT maquina, SUM(costo_hora) FROM produccion GROUP BY maquina HAVING SUM(costo_hora) > 1000;

⬆️ MAX piezas > 150 por turno

SELECT turno, MAX(piezas) FROM produccion GROUP BY turno HAVING MAX(piezas) > 150;

🔥 Reto Final: Análisis Integral

Usa GROUP BY maquina, AVG(piezas), HAVING AVG(piezas) > 145. Combina con ORDER BY. ¿Recomendarías upgrades?
